# C4 Design System <Badge text="Multi-Brand" variant="tip" />

Engine de diagramas tematizável. O componente `MermaidDiagram` injeta automaticamente o tema da marca atual — basta trocar `brand.ts` para atualizar todos os diagramas do projeto.

---

## Como funciona

```
brand.ts → c4-theme.ts → MermaidDiagram.astro → diagrama renderizado
```

1. `brand.ts` define os tokens de cor da marca
2. `c4-theme.ts` deriva `classDef` e `themeVariables` a partir desses tokens
3. `MermaidDiagram.astro` chama `injectTheme(code)` no frontmatter — injeta `%%{init}%%` e `classDef` automaticamente
4. O diagrama é renderizado com a identidade visual correta

**Os arquivos `.mdx` não precisam de `classDef` nem de `%%{init}%%`.**

---

## Classes disponíveis

| Classe | Derivada de | Uso semântico |
|--------|-------------|---------------|
| `:::system` | `surface` + `accent` | Núcleo, AI Engine, sistema principal |
| `:::container` | `card` + `accent` | Apps, APIs, workers, serviços |
| `:::database` | `bg` + `muted` | Bancos de dados, stores, logs |
| `:::decision` | `surface` + `accentHigh` | Filas, roteadores, decisões |
| `:::external` | `bg` + `border` | Usuários humanos, sistemas externos |

## Como escrever um diagrama

```mdx
import MermaidDiagram from '../../../../components/ui/MermaidDiagram.astro';

export const meuDiagrama = `
flowchart TD

A["Sistema"]:::system
B[("Banco")]:::database
C{"Decisão"}:::decision
D["Externo"]:::external

A --> C
C -->|ok| B
C -->|erro| D

class A system
class B database
class C decision
class D external
`;

<MermaidDiagram code={meuDiagrama} caption="Descrição do diagrama" />
```

<Aside type="tip" title="Sem classDef, sem init">
Não inclua `%%{init}%%` nem `classDef` nas strings dos diagramas. O componente injeta tudo automaticamente a partir de `brand.ts`.
</Aside>

## Trocar de marca

Para aplicar uma identidade visual diferente em **todos** os diagramas do projeto, edite apenas `src/config/brand.ts`:

```ts
colors: {
  accent:     '#sua-cor-primaria',
  accentHigh: '#sua-cor-primaria-clara',
  bg:         '#seu-fundo',
  surface:    '#sua-superficie',
  card:       '#seu-card',
  border:     '#sua-borda',
  text:       '#seu-texto',
  muted:      '#seu-texto-secundario',
}
```

O próximo build aplica a nova paleta em todos os diagramas automaticamente.

---

## Regras

1. **Nunca** incluir `classDef` nos arquivos `.mdx`
2. **Nunca** incluir `%%{init}%%` nos arquivos `.mdx`
3. **Sempre** usar `:::system`, `:::container`, `:::database`, `:::decision` ou `:::external`
4. **Sempre** reforçar com `class NomeDoNó classDef` ao final (dupla garantia)
5. **Sempre** `flowchart TD`

---

## Referência rápida

```
system    → núcleo, sistema principal      (surface / borda accent)
container → apps, APIs, workers            (card / borda accent)
database  → bancos, stores, logs           (bg / borda muted)
decision  → filas, roteadores, decisões    (surface / borda accentHigh)
external  → usuários, sistemas externos    (bg / borda border)
```
